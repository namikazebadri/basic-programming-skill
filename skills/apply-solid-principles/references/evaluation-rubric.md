# Evaluation Rubric

Use this file to assess whether the SOLID skill output is strong enough before returning it.

Score each category from 0 to 2.

## 1. Problem Diagnosis

- 0: Names a principle without defining the design pressure.
- 1: Identifies some design pain, but weakly.
- 2: States the concrete design pressure before naming the principle.

## 2. Simplicity Bias

- 0: Recommends abstraction immediately.
- 1: Mentions a smaller option without serious evaluation.
- 2: Compares the recommendation against a smaller local refactor.

## 3. SOLID Fit

- 0: Uses the wrong principle or uses one as a slogan.
- 1: Roughly maps the issue to a principle.
- 2: Clearly matches the design problem to the most relevant principle.

## 4. Trade-Off Awareness

- 0: Ignores costs.
- 1: Mentions one downside.
- 2: Explains ceremony, readability, or maintenance risks clearly.

## 5. Incremental Actionability

- 0: Suggests a large redesign.
- 1: Gives broad next steps.
- 2: Proposes a narrow, reviewable next step with test impact.

## Interpreting The Score

- 8-10: Strong output
- 5-7: Acceptable but needs sharper restraint
- 0-4: SOLID cargo-cult risk
